ld链接脚本学习（一）

第一章 链接脚本介绍

1. 什么是链接脚本

在我们使用gcc命令将一个一个输入文件变为目标文件.o的时候，当前文件是不能再内存中运行的，需要进行链接，由操作系统分配程序入口地址，经过重定位、符号解析等操作才能够运行。而链接脚本就是在链接过程中起着决定性作用。可以这么说，链接脚本是链接的指向灯。链接脚本说你该怎么链接，链接就会按照链接脚本所说的话去做

2. 链接脚本的后缀名及其相关说明

在linux中，链接脚本后缀名为.lds，本质上是文本文件。
链接脚本控制每次链接的进行，使用链接命令语言编写。链接脚本最主要的目的是：描述如何将输入文件中的各个sections映射到输出文件中，并控制输出文件的内存布局

第二章 链接脚本语法介绍

1. 基本语法介绍

  • 写一个链接脚本相当于写一系列命令，每个命令后面跟着的要是就是参数的关键字，要不就是符号的赋值
  • 命令可以用;隔开，空格一般情况下省略
  • 字符串，如文件或格式名称一般情况下可以直接加入。如果文件名包含类似于逗号等符号，为了避免文件名被分离。使用双引号，将文件名放到双引号中，注意不能再文件名中使用双引号
  • 可以使用类似于C语言的注释符号 /**/

2. 详细介绍

命令 说明
ENTRY(symbol) 设置入口点，即程序执行的第一条指令
链接器依次通过下述方法来进行设置，只要成功立即停止：
-e 入口命令选项（译注：-e是链接器的参数）
链接脚本中的ENTRY (symbol) 命令
如果符号start定义了，那就是start符号的值
如果存在.text节，那就是.text节的第一个字节的地址
地址0
SECTIONS 该命令告诉链接器如何将输入节映射成输出节以及输出节怎样放进内存
其格式为：
**SECTIONS
{
sections-command
sections-command
}
**其中sections-command可以由以下4项组成：
1. ENTRY命令
2. 符号赋值语句
3. 一个输出节描述
4. 一个overlay描述
INCLUDE filename 在该位置包含名为filename的链接脚本
INPUT (file， file， ….) 链接器在链接过程中包含file名字的文件
GROUP(file，file，…) 类似于INPUT，不同的是命名的文件都应该是归档文件，并且它们会被重复搜索，直到没有新的未定义的引用被创建。
OUTPUT(filename) 给输出文件命名为filename
SEARCH_DIR (path) 将路径path添加到’ ld ‘查找归档库的路径列表中
STARTUP ( filename ) 就像’INPUT 命令一样，除了文件名将成为第一个被链接的输入文件，就像它是在命令行上首先指定的一样。
OUTPUT_FORMAT 给输出文件指定BFD格式的名称
PROVIDE( symbol = expression ) 在目标文件内被引用，但没有在任何目标文件内被定义的符号

2.1 输出节

2.1.1 输出节描述

多数输出节不使用大部分可选的节属性。“SECTION”周围的空格是必需的，这样SECTION名是明确的。冒号和花括号也是必需的。换行符和其他空白是可选的。

每个output-sections-command 命令可以是如下：

  • 一个符号赋值
  • 一个输入节描述
  • 要直接包含的数据值
  • 一个特殊的输出部分关键字

2.1.2 输出节名称

输出节的名称是’ section ‘。’ section ‘必须满足输出格式的约束。在只支持有限数量的节的输出文件格式中，例如’ a.out ‘，名称必须是该格式支持的名称之一(例如’ a.out ‘，只允许’ .text ‘， ‘ .data ‘或’ .bss ‘)。如果输出格式支持任意数量的节，但是使用数字而不是名称(就像Oasys的情况一样)，则名称应该作为引用的数字字符串提供。section名称可以由任何字符序列组成，但是包含逗号等任何不寻常字符的名称必须用引号括起来。输出节名’ /DISCARD/ ‘是特殊的（丢弃输入文件的节）

2.1.3 输出节地址

“address”是输出节的VMA(虚拟内存地址)的表达式。如果你没有提供’ address ‘，链接器将基于’ region ‘(如果存在)设置它，或者以其他方式基于位置计数器的当前值。

如果你提供了’ address ‘，输出节的地址将被精确地设置为该规范。如果你既不提供’ address ‘也不提供’ region ‘，那么输出节的地址将被设置为符合输出节的对齐要求的位置计数器的当前值。

输出节的对齐要求是输出节中包含的所有输入节中最严格的对齐方式。（译注：比如，一个输出节包括2个输入节，一个输入节是2字节对齐，一个输入节是4字节对齐，那么这个输出节就按照最严格的4字节对齐）
